Les lignes noires représentents les bus de données et les lignes oranges les bus d'adresses. Je n'a pas représenté les signaux qui sortent de l'unité de contrôle (UC) qui seraient rattachés à tous les autres composants.

Description des composants du processeur :

UAL 8 bits unité arithmétique et logique effectue des opérations sur des nombres binaires
A 8 bits accumulateur stocke le résultat de l'UAL
B
MEM 64k mémoire entrée 16 bits pour l'adresse et interface 8 bits pour écrire et lire des données
FL 8bits registre des flags ("drapeaux") indique les caractéristiques du résultat d'une opération effectuée par l'UAL
OP 16bits registre d'instruction contient l'instruction actuellement exécutée
UC unité de contrôle : décode le registre OP et contrôle chaque composant du processeur pour exécuter l'instruction
PC 16bits compteur ordinal contient l'adresse de l'instruction actuellement exécutée
JH 8bits registre supérieur de saut l'octet supérieur de l'adresse de saut
MH 8bits registre supérieur de mémoire l'octet supérieur d'une adresse mémoire
SP 16bits pointeur stack ("pile") donne l'adresse du sommet du stack
MUX multiplexeur permet de choisir le signal de sortie parmi les signaux d'entrée

Registre des flags. Il est utilisé lorsque l'on fait un saut conditionnel : par exemple si l'on ordonne de sauter uniquement si le flag "zéro" est mis l'unité de contrôle va regarder l'état de ce registre. Le flag "interruption" est un peu spécial puisqu'il indique si le processeur est "interruompu", je reviendrai sur cette notion avec les interfaces (I/O). Les 4 bits non utilisés pourraient éventuellement être tout de même utilisés par un programme, l'unité de contrôle ne devrait alors ne pas y toucher.
